# 算法期末备考 -- 解题套路总结

## 一、拿到一道题的行动路线

```
1. 读题
2. 它属于哪个范式? (分治 / DP / 贪心 / 堆)
3. 写出函数签名 (先定义输入输出)
4. 手写核心逻辑 (先写最简版本, 再优化)
5. 检查边界 (空 / n=0 / n=1)
```

---

## 二、分治 (Divide & Conquer)

### 三步走框架

1. **分解 (Divide)**: 确定分界点, 分成子问题
2. **解决 (Conquer)**: 递归处理子问题
3. **合并 (Combine)**: 子问题的结果组合成最终答案

### 函数签名设计法则

```
返回类型 solve(数据, 左边界, 右边界)
```

通用模式: `solve(arr, l, r)` 处理 `[l, r)` 区间.

- 为什么用 `[l, r)`? `r - l = 长度`, 空区间 `l == r`, 不易 off-by-one.
- 子问题范围缩小: `[l, mid)` + `[mid, r)`
- 递归基: `r - l <= 1` (空或只有一个元素)

### 记忆化 (Memoization)

```
返回值/void solve(参数列表, memo数组)
  if (memo[n] != 0) return memo[n];
  memo[n] = solve(n-1) + solve(n-2);
  return memo[n];
```

### 分治核心规律

子问题 = 缩小数据范围 + 同样的处理方法.

## 三、动态规划 (Dynamic Programming)

### 四步走框架

1. **状态定义 (State)**: `dp[i][j]` 表示什么
2. **转移方程 (Transition)**: `dp[i][j]` 从哪些值算出来
3. **初始化 (Base Case)**: 边界值是多少
4. **遍历顺序 (Order)**: 保证计算时依赖项已算好

### 状态定义法则

问三个问题:

| 问题 | 答案影响 |
|------|----------|
| 问题有几个变量? | 几维 DP |
| 变量是什么类型? | 下标用 i/j, 容量用 w, 个数用 k |
| 答案是什么类型? | 最小值/最大值 -> int; 方案数 -> long/mod; 是否可行 -> boolean |

### 常见模式

| 问题类型 | 状态定义 | 维度 |
|----------|----------|------|
| 线性序列 (爬楼梯, 装配线) | `dp[i]` 处理到第 i 个位置的最优值 | 1D |
| 两个序列比较 (LCS) | `dp[i][j]` X前i和Y前j的答案 | 2D |
| 区间问题 (LPS) | `dp[i][j]` 区间 [i,j] 的答案 | 2D |
| 背包问题 | `dp[i][w]` 前i个物品, 容量w的最大价值 | 2D (可优化到1D) |

### 转移方程速成

```
dp[i] = 最优(dp[i-1] + 选择A的代价, dp[i-2] + 选择B的代价, ...)
```

对每一步, 列出所有可能的上一步状态, 取最优.

### 遍历顺序速查表

| DP 类型 | 遍历顺序 | 原因 |
|---------|----------|------|
| 线性 1D | i 从小到大 | 依赖 i-1 |
| 序列比较 2D (LCS) | i 从 1..m, j 从 1..n | 依赖左上/上/左 |
| 区间 DP (LPS) | i 从大到小, j 从小到大 | 依赖左下/下/左 |
| 0-1 背包 1D | w 从大到小 | 每个物品只用一次 |
| 完全背包 1D | w 从小到大 | 每个物品可无限用 |

## 四、堆 (Heap)

堆 = 数组 + 两个操作.

```
数组 h[1..size]
父 i, 左子 2i, 右子 2i+1
```

| 操作 | 逻辑 | 用途 |
|------|------|------|
| `maxHeapify(h, i, size)` | i 的位置违反堆性质, 把它往下调整 | 修正单个节点 |
| `buildMaxHeap(h, size)` | 从 `size/2` 到 1 依次 maxHeapify | 构建初始堆 |

大顶堆: 父 >= 子; 小顶堆: 父 <= 子.

---

## 五、贪心 (Greedy)

### 三步走

1. **排序**: 按某个标准排 (性价比 / 截止时间 / 右端点)
2. **顺序选取**: 从排好序的序列里依次做决策
3. **不可回头**: 选了就不能改

### 区分贪心 vs DP

| 特征 | 贪心 | DP |
|------|------|----|
| 物品能否拆分 | 可拆 -> 分数背包 | 不可拆 -> 01 背包 |
| 当前选择是否影响未来 | 不影响 | 影响 |
| 实现复杂度 | O(n log n) 排序 | O(n^2) 或更高 |

### 6.5 栈 (Stack) -- 不用 Stack 类, 用 ArrayDeque

```java
import java.util.ArrayDeque;
import java.util.Deque;

Deque<Integer> stack = new ArrayDeque<>();
stack.push(x);     // 入栈
int x = stack.pop();      // 出栈
int x = stack.peek();     // 看栈顶不弹出
boolean empty = stack.isEmpty();
int size = stack.size();
```

### 6.6 队列 (Queue)

```java
import java.util.Queue;
import java.util.LinkedList;

Queue<Integer> q = new LinkedList<>();
q.offer(x);     // 入队 (推荐, 不抛异常)
q.add(x);       // 入队 (容量满时抛异常)
int x = q.poll();       // 出队 (空返回 null)
int x = q.peek();       // 看队首 (空返回 null)
boolean empty = q.isEmpty();
```

### 6.7 优先队列 (堆)

```java
import java.util.PriorityQueue;

// 小顶堆 (默认)
PriorityQueue<Integer> pq = new PriorityQueue<>();

// 大顶堆
PriorityQueue<Integer> pq = new PriorityQueue<>((a, b) -> b - a);
// 或
PriorityQueue<Integer> pq = new PriorityQueue<>(Collections.reverseOrder());

// 存自定义类型
PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[0] - b[0]);

pq.offer(x);         // 插入
int x = pq.poll();   // 取出堆顶并删除
int x = pq.peek();   // 看堆顶不删除
int size = pq.size();
```

### 6.8 HashMap

```java
for (Map.Entry<K, V> e : map.entrySet()) {

```
